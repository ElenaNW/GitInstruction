# Работа с Git

## 1. Проверка наличия установленного Git
В терминале выполнить команду `git vertion`. 
Если Git установлен, появится сообщение с информацией о версии программы. 
Иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию Git с сайта https://git-scm.downloads. Устанавливаем с настройками по умолчанию. 
## 3. Первые шаги. Создаем папку. Знакомство с терминалом и другими окнами VSCode.
* На компьютере создаём папку - место для хранения файлов, изменения в которых будут отслеживаться.
* В VSCode открываем эту папку (File - Open Folder)
```
Рабочее пространство VSCode делится на несколько зон. 
 Чтобы открыть Терминал (окно, в котором будут вводиться команды, показываться результаты и т.д.) выберите в проводнике View - Terminal.
 Верхнюю часть экрана, делим на 2 при помощи кнопки Open Preview to the Side (правый верхний угол экрана). В левой части мы видим рабочий файл, в правой конечный результат. 
```

## 4. Настройка Git
При первом использовании Git необходимо представиться. 
Для этого нужно выполнить в терминале 2 команды 
```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```
## 5. Инициализация Git. Создание нового файла. 
Далее необходимо инициализировать Git, введя в строку в терминале команду `git init`. Эта команда создаёт локальный репозиторий.
При этом в папке появится скрытая папка .git
Создаём новый файл:
1. В левом верхнем углу (EXPLORER) нажимаем на значек `New File`
2. Вводим название файла и его расширение.
3. Для того, чтобы новый файл стал отслеживаться в Git, в терминале вводим команду `git add название файла` (название файла поможет ввести кнопка Tab)
 
## 6. Команды Git для работы с файлом.
* `git commmit -m "комментарий к фиксации"` сохраняет (фиксирует) изменения.
* `git status` показывает текущий статус файла (на какой ветке находится, вносились ли изменения после фиксации)
* `git diff` показывает разницу между текущей и зафиксированной версией
* `git log` выводит список всех коммитов (фиксаций) в хронологическом порядке
* `git checkout`позволяет перемещаться между сохранениями, задав их номер, или между ветками. Не забывайте вернуться в нужную версию сохранения, текущую версию или в нужную ветку.
* `git commit -a` переход во встроенный текстовый редактор, где можно ввести текст комментария коммита. Нажмите `i` чтобы перейти в режим редактирования текста, клавишу `Esc` что бы сохранить редакцию, `:wq` чтобы выйти из редактора.
* `git commit -am "комментарий к фиксации"` одновременное добавление и фиксация 
## 7. Игнорирование файлов.
Для того, чтобы исключить из отслеживания в репозитории определённые файлы или папки, необходимо создать файл ***.gitignore*** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.

## 8. Создание веток в Git.
Ветка в Git - это простой перемещаемый указатель на один из коммитов, обычно последний в цепочке коммитов. 
По умолчанию имя основной ветки в Git - *master*.

Создать ветку можно командой 
```
git branch <имя новой ветки>
```
В результате создаётся новый указатель на текущий коммит.
Список веток в репозитории можно посмотреть с помощью команды 
```
git branch
```
Текущая ветка будет отмечена звёздочкой ***master**. 
Так же ветку можно создать при помощи команд `git branch -b <имя создаваемой ветки>` и `git switch - c <имя создаваемой ветки>` . В этом случае произойдёт одновременное создание и переход на новую ветку.

В названии новой ветки не должно быть пробелов.

## 9. Переход с одной ветки на другую.
Для перехода с текущей ветки на другую нужно воспользоваться командой
```
 git checkout <имя ветки, на которую нужно перейти>
```
Перед переходом не забудьте сделать коммит.
## 10. Слияние веток и разрешение конфликтов.
Перед тем, как сделать слияние, нужно зафиксировать изменения. Далее, для слияния выбранной ветки с текущей нужно выполнить команду:
```
git merge <название выбранной ветки>
```
Если была изменена одна и та же часть файла в обеих ветках, то может возникнуть конфликт, который потребует участия пользователя. Это будет выглядеть так: в рабочем файле появятся 2 поля (не во всех версиях VSCode), зеленого и синего цветов. Одно из них будет с вариантом текста из текущей ветки, второе из той, которую мы сливаем с текущей. Над ними VSCode предлодит 4 вариант решения конфликта:
1. Принять вариант из текущей ветки
2. Принять вариант из ветки, которую мы сливаем с текущей
3. Принять оба варианта
4. Просмотреть разницу между вариантами (и затем принять решение).
После выбора нужного варианта, текст можно редактировать далее. 
Важно! После разрешения конфликта необходимо сохранить результат, введя команды
```
git add <имя файла>, а затем git commit -m "комментарий к фиксации"
```
## 11. Удаление ветки.
Если какая-то из веток оказалась ненужной, её можно удалить при помощи команды
```
git branch <имя ветки>
```
Но если эта ветка была не слита, а изменения в ней не были зафиксированны, при удалении появится ошибка. Если эту ветку всё равно надо удалить, учитывая эти факты, можно воспользоваться командой
```
git branch -D <имя ветки>
```
## 12. Язык разметки Markdown.
1. Чтобы сделать текст курсивом, нужно обрамить его * или _  *Мороз и солнце*  _день чудесный_
2. Сделать текст полужирным поможет обрамление ** или __ **Ещё ты дремлешь,** __друг прелестный -__  
3. Зачёркнутый текст можно получить при помощи ~~ ~~пора, красавица, проснись:~~ 
4. Пункты 1-3 можно легко комбинировать __*Открой* сомкнуты негой взоры__
5. _Навстречу северной **Авроры**,_
6. Выделить цветом поможет обрамление апострофом `Звездою севера явись!` 
7. Выделить пункты точками поможет *, а пронумеровать их легко при помощи цифр.
8. Заголовок выделяется при помощи # перед названием, заголовок поменьше при помощи ##

## 14. Работа с удалёнными репозиториями
1. Создать аккаунт на GitHub (`New repository`, дать имя репозиторию, `Create repository`)
2. GitHub предложит 3 варианта дальнейших действий с командами (использовать подходящий):
* Создать новый рабочий репозиторий через терминал и начать с ним работать
* Привязать существующий репозиторий к удалённому
* Импортировать код из другого репозитория
3. Использовать команды из подходящего варианта, связать удалённый и локальный репозитории (по умолчанию GitHub предлагает главную ветку **main**, поэтому ветку **master** нужно переименовать).
4. Перейти в удалённый репозиторий на GitHub и обновить страницу. Появится поле с файлом локального репозитория. 
5. Нажав на значек карандаша с этим файлом можно работать и вносиьт изменения. Сохранить изменения нужно при помощи кнопки `Commit change.`
6. Перенести изменения из удалённого репозитория в локальный:
* перейти в локальный репозиторий
* ввести команду `git pull`
* если появится конфликт, разрешить его так же, как конфликт между содержанием веток.
* сделать коммит слияния `git commit -am <текст комментария>`
7. отправить изменения с локального репозитория в удалённый комнадой `git push`
8. Чтобы передать свой репозиторий через GitHub, нужно после его загрузки скопировать ссылку из кнопки **Code** и поделиться ей. 

## 15. Запрос на вливание своих изменений в чужой репозиторий
1. Зайти на GitHub, найти нужный репозиторий через поиск или по ссылке
2. Сделать копию этого репозитория с помощью кнопки **Fork** 
```
если нужны все ветки репозитория, нужно снять галочку с "Copy the master branch only" 
```
Именно с этой копией и нужно работать.
3. Скопировать ссылку на этот репозиторий (Кнопка `Code`, затем нажать `Copy` справа от ссылки) 
4. Перейти в VSCode, открыть нужную папку *(Внимание! Эта папка не должна быть репозиторием)*, ввести в терминале команду git clone <вставить скопированную ссылку> <написать название>
5. Открыть папку репозитория в терминале (клик правой кнопкой на имя папки в терминале, выбрать "открыть во встроенном терминале""). Перейти в него, нажав на папку. 
6. Создать отдельную ветку , предварительно посмотрев, какие ветки уже существуют. Перейти на неё.
7. Создать изменения, добавить их в отслеживание, сделать коммит.
8. Создать новую ветку в удалённом репозитории и связать её с веткой в локальном репозитории при помощи команды `git push --set-upstream <название удаленного репозитория, по умолчанию origin> <название ветки локального репозитория>`
9. Перейти в удалённый репозиторий и обновить его. Сделать pull request.  Можно оставить комментарий для автора репозитория.
20. Автор может выполнить слияние (кнопка **Merge**) либо написать сообщение отправителю.

## 16. Полезные примечания.
* Прежде чем перейти к другому коммиту, зафиксируйте изменения в текущей версии
* Не забывайте сделать команду `git add название файла` перед командой `git commit`
* Проверка текущего статуса файла никогда не будет лишней  
* Не забывайте менять раскладку клавиатуры с русского на английский и наоборот.
* в своих сообщениях Git удалённые и неотслеживаемые строки выделяет красным цветом, а добавленные и сохраненные зелёным.
* Книга по Git на официальном сайте https://git-scm.com/book/ru/v2
